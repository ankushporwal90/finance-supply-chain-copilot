# Development Phases

This repo will be built in deliberate phases so each concept is understandable.

## Phase 1: Structure and Setup

Goal: create a professional repository skeleton with configuration, docs, and a minimal Streamlit shell.

Key concepts:

- Separation of concerns
- Environment variables
- Local free storage
- Basic app entrypoint

## Phase 2: Environment Setup

Goal: confirm local dependencies, `.env`, Streamlit startup, and basic tests.

Key concepts:

- Virtual environments
- Dependency management
- Free API keys
- Reproducibility

## Phase 3: Core App Shell

Goal: improve navigation, dashboard layout, empty states, and reusable UI patterns.

## Phase 4: Finance Dashboard

Goal: stock lookup, live snapshot, historical chart, company overview, and KPI cards.

## Phase 5: RAG Pipeline

Goal: PDF upload, text extraction, chunking, embeddings, ChromaDB storage, retrieval, and citations.

## Phase 6: Chat System

Goal: grounded chat with reports, financial analysis prompts, and source-aware answers.

## Phase 7: Memory and State

Goal: conversation continuity, selected company context, uploaded document context, and optional SQLite persistence.

## Phase 8: Alerts Engine

Goal: price thresholds, percentage-change alerts, volatility alerts, and report sentiment alerts.

## Phase 9: Deployment

Goal: deploy on Streamlit Community Cloud for free.

## Phase 10: Refactoring and Documentation

Goal: polish README, tests, architecture docs, portfolio framing, and interview talking points.
